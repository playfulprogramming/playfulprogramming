> I don't like too complex configurations, I like the idea of ‚Äã‚Äãbeing able to be productive in any environment and with the standard configuration, but for these 5 VSCode extensions, I made an exception.

### [Import cost](https://marketplace.visualstudio.com/items?itemName=wix.vscode-import-cost)

![ImportCost](./7lgcqz2ydfb3bqtdk4qh.png)

#### In action

![ImportCostFlow](https://citw.dev/_next/image?url=%2fposts%2fimport-cost%2f1quov3TFpgG2ur7myCLGtsA.gif\&w=1080\&q=75)

Thanks to this plugin I can immediately see how much an import costs me and keep my bundle size under control.

### [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode)

![Prettier](./9h85qerbj7lg2br6n6bj.png)

#### In action

![PrettierInAction](./g94upwd6ixw6ie66krhe.gif)

Prettier is a code formatter that enforces a uniform style, if you have multiple people working on a single project, this is definitely a must have.

### [Peacock](https://marketplace.visualstudio.com/items?itemName=johnpapa.vscode-peacock)

![Peacock](./0pcybqnd5avzl9zvdxja.png)

#### In action

![PeacockInAction](./hero.png)

Peacock lets you change the color of your Visual Studio Code environment. You can set a different color for each project to quickly identify which application you're working on.

### [Version Lens](https://marketplace.visualstudio.com/items?itemName=pflannery.vscode-versionlens)

![Vlens](./5kbqb44lhlav5s88ql1a.png)

#### In action

![VLensInAction](./n7c8icp70ttsednuha12.gif)

This extension displays information by reading package.json and allows you to update dependencies.

### [GitLens](https://marketplace.visualstudio.com/items?itemName=eamodio.gitlens)

![GitLens](./lq1pzunvqbzuzgg6tqxs.png)

I find this extension very useful and *line blame annotation* feature is really useful in my daily job.

![blame](./ooa2ff01ek1nnghjx95z.png)
